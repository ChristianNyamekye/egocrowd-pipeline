<task id="2" name="Replace binary finger control with distance-based pre-shaping">
## What
Replace the binary `FINGER_CLOSED if want_grip else FINGER_OPEN` finger control with a three-phase system: distance-based pre-shaping during approach, full closure on grasp, and smooth return to open on release.

## Why
OUT-03 requires fingers to visibly begin closing during descent approach, proportional to palm-to-block distance. The current binary switch looks robotic. Distance-based pre-shaping creates natural anticipatory finger curling.

## How

**Step 1: Replace the finger control block (lines 422-424).**

Find:
```python
        # Finger control
        target_ctrl = FINGER_CLOSED if want_grip else FINGER_OPEN
        finger_ctrl += (target_ctrl - finger_ctrl) * 0.25
        data.ctrl[HAND_CTRL_START:HAND_CTRL_START+7] = finger_ctrl
```

Replace with:
```python
        # Finger control with distance-based pre-shaping (OUT-03)
        if want_grip:
            # Full closure for data-driven grasp
            finger_target = FINGER_CLOSED
            blend_rate = 0.25
        else:
            # Distance-based pre-shaping: curl fingers as palm approaches block
            min_block_dist = min(
                np.linalg.norm(data.xpos[bid] - pc) for bid in obj_body_ids
            )
            preshape_t = np.clip(
                1.0 - (min_block_dist - PRESHAPE_DIST_FULL) / (PRESHAPE_DIST_START - PRESHAPE_DIST_FULL),
                0.0, 1.0
            )
            preshape_t = preshape_t * preshape_t  # quadratic ease-in
            if preshape_t > 0.01:
                finger_target = FINGER_OPEN + (FINGER_PRESHAPE - FINGER_OPEN) * preshape_t
                blend_rate = 0.15  # slower blend for anticipatory shaping
            else:
                finger_target = FINGER_OPEN
                blend_rate = 0.20
        finger_ctrl += (finger_target - finger_ctrl) * blend_rate
        data.ctrl[HAND_CTRL_START:HAND_CTRL_START+7] = finger_ctrl
```

- The `pc` variable (palm center) is already computed earlier in the frame loop (line 362: `pc = palm_center(model, data)`) and is available here.
- `obj_body_ids` is already available (line 178).
- The `preshape_t` quadratic ease-in means fingers curl slowly at first, then accelerate as the hand gets closer.
- When `want_grip=True`, the existing fast blend (0.25) still applies for responsive grasp closure.
- When `want_grip=False` and hand is far from blocks, fingers return to open with medium blend (0.20).

<task id="3" name="Add collision exclusion during kinematic hold">
## What
Disable collision for the grasped block during kinematic hold by setting its contype/conaffinity to 0 when the grasp attaches, and restoring them to 1 when released.

## Why
OUT-04 requires no visible finger-block interpenetration during kinematic hold. Currently, finger collision geoms and block collision geoms both have contype=1/conaffinity=1, so MuJoCo computes contact forces that fight the kinematic attachment. Disabling block collision during hold eliminates this conflict. The block position is kinematically controlled during hold anyway, so collision is not needed.

## How

**Step 1: Disable grasped block collision inside the ATTACH block.**

Find the end of the ATTACH section (line 390):

```python

                        model.geom_conaffinity[gi] = 0
```
